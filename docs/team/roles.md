# 팀 역할 분담

## 현재 팀 구성 가정

- 개발자 2명
  - 팀장 1명
  - 구현 보조/협업 개발자 1명
- 비개발자 3명
  - 게임 기획/콘텐츠 담당
  - 아트/UI 에셋 담당
  - 사운드/테스트/운영 보조 담당

이 문서는 **이름 배정표**라기보다 현재 팀 규모에서의 **책임 분담 기준**을 설명한다.

## 역할

| 역할 | 주요 책임 | 기본 오너 |
|------|-----------|-----------|
| 팀장 / 테크 리드 | MVP 범위 확정, 아키텍처, 수직 슬라이스 구현, 우선순위 결정 | 개발자 1 |
| 협업 개발자 | 저위험 구현, 디버깅, 테스트, 반복 작업, 리팩터링 보조 | 개발자 2 |
| 게임 기획 / 콘텐츠 | 규칙 구체화, 무기/맵 기획, 밸런스 피드백, 플레이 테스트 체크리스트 | 비개발자 1 |
| 아트 / UI 에셋 | 캐릭터, 무기, 배경, HUD 시안 및 규격 정리 | 비개발자 2 |
| 사운드 / QA / 운영 보조 | 효과음, BGM 초안, 버그 재현 기록, 테스트 세션 운영 | 비개발자 3 |

## 역할 분배 원칙

1. 팀장은 초기에 **핵심 구조와 기준 계약**을 직접 만든다.
2. 두 번째 개발자는 핵심 아키텍처를 병렬로 뜯기보다, 분리 가능한 작업을 맡는다.
3. 비개발자는 "나중에 투입"이 아니라, 기획/에셋/테스트 자산을 먼저 만든다.
4. 한 작업에는 반드시 오너 1명이 있어야 한다.
5. 리뷰어와 오너는 가능하면 분리한다.

## 현재 추천 운영 방식

초기 v0 단계에서는 아래처럼 운영한다.

- 팀장: Portal/Game/Server를 관통하는 최소 플레이 가능 버전 구축
- 협업 개발자: 문서 기반 구현 보조, 테스트, 버그 수정, 보조 기능
- 비개발자: 무기표, 맵 아이디어, HUD 텍스트, 에셋 목록, 테스트 시나리오 제작

자세한 운영 규칙은 [협업 규칙](collaboration-rules.md)을 따른다.
