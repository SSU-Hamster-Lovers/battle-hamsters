# 미니 스펙 — 스폰 동작 구분 1차 (floating / falling)

## 작업명

스폰 동작 구분 1차

## 목표

무기/아이템이 모두 같은 방식으로만 나타나는 상태에서 벗어나,
공중에 떠 있는 스폰과 중력에 영향을 받아 떨어지는 스폰을 구분한다.

## 이번 범위

- `spawnStyle`의 런타임 의미를 실제 동작으로 연결
- `floating` 성격 스폰과 `falling` 성격 스폰 구분
- 무기와 아이템 양쪽에 적용 가능한 기준 정의
- 맵 데이터에서 손으로 조정 가능한 수준까지 정리

- 바람/컨베이어 같은 복잡한 이동 물리

## 핵심 결정

### 1. 맵 에디터보다 데이터 스키마가 먼저다

- 1차는 손으로 JSON을 수정해도 충분하다.
- 먼저 맵 포맷과 런타임 해석 규칙을 안정화한다.

### 2. `spawnStyle`을 실제 의미로 사용한다

- 예:
  - `fade_in` → floating
  - `airdrop` → falling
- 필요하면 후속 단계에서 별도 동작 필드를 추가한다.

### 3. 무기와 아이템은 같은 개념을 공유하되 세부 연출은 분리 가능하게 둔다

- 둘 다 “떠 있음 / 떨어짐”이라는 공통 개념은 공유
- 시각 연출은 후속에서 따로 세분화 가능

## 완료 조건

- 맵 데이터가 떠 있는 스폰과 떨어지는 스폰을 표현할 수 있다.
- 서버/클라이언트가 해당 차이를 같은 기준으로 해석할 수 있다.
- 에디터는 아직 없어도 런타임 검증이 가능하다.

## 검증 방법

- 문서 기준 정합성 검토
- 이후 구현 시 floating / falling 스폰을 각각 수동 테스트
