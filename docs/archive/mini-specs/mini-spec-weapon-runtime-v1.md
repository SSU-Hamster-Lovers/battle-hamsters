# 미니 스펙 — 무기 시스템 1차 (pickup / 장착 / 발사 / 넉백 / 자기 반동)

## 작업명

무기 시스템 1차

## 목표

기본 플랫폼 이동 위에 **월드 무기 획득 → 장착 → 발사 → 자원 소모 → 무기 종료**의 첫 전투 루프를 올린다.
이 단계에서는 상대 넉백뿐 아니라 **발사한 자기 자신이 밀리는 반동(self recoil)**도 함께 설계 기준에 포함한다.

## 이번 범위

- 월드 `weaponSpawns`를 실제 런타임 상태로 반영한다.
- 플레이어가 바닥 무기를 획득/교체/드롭할 수 있게 한다.
- 기본 보급 무기 1종(`Acorn Blaster`)의 실제 발사를 구현한다.
- 발사 시 **상대 넉백**과 **자기 반동**을 함께 처리한다.
- 탄창형 자원 소모와 빈 무기 폐기 규칙을 반영한다.
- 관련 스냅샷/프로토콜 필드를 실제 런타임 의미에 맞게 정리한다.

## 핵심 결정

### 1. pickup만 따로 떼지 않고 “무기 시스템 1차”로 묶는다

- pickup만 먼저 만들면 발사/소모/폐기 규칙이 다시 따로 흔들릴 수 있다.
- 따라서 이번 단계는 **획득 + 장착 + 발사 + 자원 소모 + 폐기**를 한 번에 묶는다.

### 2. 자기 반동은 넉백과 별도 축으로 둔다

- `knockback`: 상대에게 가는 힘
- `selfRecoil`: 발사자 자신에게 가는 힘
- 둘은 독립적으로 조절 가능해야 한다.

### 3. 자기 반동은 무기별 데이터로 조절 가능해야 한다

- 기본 세기
- 방향 기준
- 방향 오차
- 공중/지상 보정
  을 무기별로 다르게 둘 수 있어야 한다.

예시:

- 권총: 낮은 `selfRecoil`
- 핸드캐논/산탄총: 높은 `knockback` + 높은 `selfRecoil`

### 4. 자기 반동 방향은 기본적으로 에임 반대 방향을 따른다

- 기본 후보: 현재 에임 방향의 `180도` 반대
- 다만 무기별로:
  - 약간 위로 뜨게
  - 약간 랜덤 오차를 주게
  - 완전 고정 방향으로 튕기게
    조절할 수 있어야 한다.

### 5. 랜덤 반동 오차는 “있을 수 있음”으로 열되 과하지 않게 시작한다

- 초기 구현은 너무 큰 난수보다 **작은 각도 jitter**만 허용하는 쪽이 좋다.
- 예능성은 살리되 제어 불가능한 수준은 피한다.

## 1차 대상 무기

### Paws

- 무한 기본 무기
- 기존처럼 fallback 무기로 유지
- 이번 단계에서는 자기 반동 없음

### Acorn Blaster

- 첫 실제 보급 무기
- `hitscan`
- `single`
- `magazine`
- pickup / 발사 / 소모 / 폐기 / 자기 반동 기준 무기 역할

## 자기 반동 데이터 초안

장기적으로는 아래와 같은 필드가 가능해야 한다.

| 필드                         | 설명                               |
| ---------------------------- | ---------------------------------- |
| `selfRecoilForce`            | 발사자에게 적용되는 반동 기본 세기 |
| `selfRecoilAngleDeg`         | 에임 반대 방향 기준 추가 각도 보정 |
| `selfRecoilAngleJitterDeg`   | 발사마다 줄 수 있는 각도 오차      |
| `selfRecoilGroundMultiplier` | 지상 반동 배수                     |
| `selfRecoilAirMultiplier`    | 공중 반동 배수                     |

## 완료 조건

- 월드 무기 스폰이 실제 pickup 가능한 상태가 된다.
- 플레이어가 보급 무기를 들고 발사할 수 있다.
- 발사 시 상대 넉백과 자기 반동이 함께 적용된다.
- 탄이 0이 되면 무기가 폐기되고 기본 무기로 되돌아간다.
- current-implementation 문서가 새 전투 상태를 반영한다.

## 검증 방법

- 서버 테스트 또는 최소 통합 검증으로 pickup / 발사 / 자원 소모 / 폐기 확인
- 2클라이언트 수동 테스트로 피격/넉백 확인
- 자기 반동이 지상/공중에서 의도한 방향으로 적용되는지 확인
- 빈 탄창이 되면 즉시 기본 무기로 돌아가는지 확인
