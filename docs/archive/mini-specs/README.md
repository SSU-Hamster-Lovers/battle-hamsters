# 완료된 미니 스펙 아카이브

이 디렉터리는 **이미 구현이 끝났거나 기준선 역할을 마친 작업용 미니 스펙**을 보관한다.

## 목적

- 과거 작업 계획을 나중에 다시 추적할 수 있게 한다.
- 왜 이런 순서로 작업했는지 문맥을 복원할 수 있게 한다.
- 장기 설계 문서와 임시 작업 문서를 분리한다.

## 운영 규칙

1. **진행 중 / 다음 작업용 미니 스펙**은 `docs/technical/mini-spec-*.md`에 둔다.
2. 작업이 끝나면 미니 스펙을 바로 삭제하지 말고 **develop에 merge된 뒤** `docs/archive/mini-specs/`로 이동한다.
3. 아카이브로 이동하기 전에 핵심 내용은 아래 문서에 반영해야 한다.
   - `docs/technical/current-implementation.md`
   - 관련 설계 문서 (`docs/game-design/*`, `docs/technical/*`)
   - 필요 시 `docs/ROADMAP.md`
4. 아카이브 문서는 **현재 기준 문서가 아니다.**
   - 현재 동작/계약은 항상 최신 스펙 문서를 우선한다.
5. 과거 미니 스펙은 **구현 기록을 보조하는 참고 자료**로 남긴다.
   - 최종 diff/세부 변경 이력은 git/PR이 정본이다.

## 권장 네이밍

- 원본 파일명을 유지한다.
- 필요하면 파일 상단에 `Archived` 표기를 추가한다.

## 참고

- 장기 규칙/계약은 아카이브가 아니라 **현재 스펙 문서**에 남겨야 한다.
- 아카이브는 “작업 계획 이력”, 현재 스펙 문서는 “지금 기준의 정답” 역할을 맡는다.
