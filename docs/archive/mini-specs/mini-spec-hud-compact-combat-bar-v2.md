# 미니 스펙: HUD compact combat bar v2

작업명:
Chaos Faction 2 스타일의 compact HUD 2차 정리

목표:
현재 카드형 HUD를 더 얇고 즉시 읽히는 전투 바 구조로 단순화해, 플레이필드 침범과 내부 겹침을 줄인다.

이번 범위:
- 좌/우 하단 HUD를 `얼굴 + 긴 HP 바 + 생명 아이콘 + 작은 무기 아이콘` 중심의 전투 바 형태로 재구성
- `HP`, `Life` 같은 라벨 텍스트를 제거하고 숫자/아이콘 위주로 정리
- 중앙 타이머 패널 폭을 줄이고 하단 바 전체 밀도를 다시 맞춤
- Free Play 우측 대상 선택 규칙은 기존의 `최근 공격 대상 -> 킬 최다 상대`를 유지
- 우상단 알림 로그와 좌상단 `WS/ping` 표시는 유지하되, 하단 바와 시각 언어를 맞춘다

건드리는 스펙/문서:
- `docs/technical/current-implementation.md`
- `docs/ROADMAP.md`
- 필요 시 `docs/technical/mini-spec-hud-polish-v1.md`

비목표:
- 팀전/8인전 전체 HUD 슬롯 설계
- 실제 atlas/spritesheet 기반 최종 HUD 아트 제작
- 알림 로그 시스템 재설계
- 무기/전투 규칙 변경

검증 방법:
- `corepack pnpm --dir apps/game typecheck`
- `corepack pnpm --dir apps/game build`
- 로컬 플레이로 1v1 / Free Play에서 하단 바가 겹치지 않는지 확인
- 1366x768 기준 플레이필드 하단 점유율과 가독성 확인

## 방향

- HUD는 `프로필 카드`보다 `전투 상태 바`에 가깝게 보이게 한다.
- 텍스트보다 색, 아이콘, 길이 차이로 정보를 먼저 읽게 만든다.
- 이름은 짧게 유지하고, 남는 공간은 HP 바 길이와 생명 수 가독성에 우선 배분한다.
- 플레이 중 시선이 중앙에서 크게 벗어나지 않도록 좌/우 바의 높이와 복잡도를 낮춘다.

## 레이아웃 초안

- 좌하단:
  - 로컬 플레이어 얼굴
  - 얼굴 우측의 긴 HP 바 1개
  - HP 바 주변에 생명 씨앗 아이콘 나열
  - 우측 끝에 작은 무기 아이콘
- 중앙 하단:
  - 타이머 또는 `FREE PLAY`
  - 기존 패널보다 폭을 줄이고 단일 정보만 유지
- 우하단:
  - 현재 대상 또는 상대 상태 바
  - 기본 규칙은 `최근 공격 대상 -> 킬 최다 상대`

## 규칙

- 세부 라벨 텍스트는 기본적으로 제거한다.
- 이름은 짧게 자르고, 잘린 경우에도 HP 바 폭을 우선 확보한다.
- HP는 숫자보다 막대 길이로 먼저 읽히게 한다.
- 생명/킬/무기 정보는 보조 정보로 취급하고, 주 정보인 얼굴/HP를 가리지 않아야 한다.
