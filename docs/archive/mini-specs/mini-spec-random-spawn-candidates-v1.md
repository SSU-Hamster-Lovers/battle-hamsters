# 미니 스펙 — 후보 지점 랜덤 스폰 1차

## 작업명

후보 지점 랜덤 스폰 1차

## 목표

무기/아이템이 항상 고정 위치에서만 나타나는 단조로운 테스트 구조에서 벗어나,
맵이 미리 정의한 안전한 후보 지점들 중 랜덤으로 스폰되게 만든다.

## 이번 범위

- 완전 랜덤 anywhere 대신 **후보 지점 랜덤** 정책 정의
- 고정 스폰과 랜덤 후보 스폰 공존 구조 정의
- 무기/아이템 모두에 적용 가능한 데이터 포맷 초안

## 건드리는 스펙/문서

- `docs/technical/data-formats.md`
- `docs/technical/current-implementation.md`
- 필요 시 `docs/ROADMAP.md`

## 비목표

- 완전 자유 좌표 랜덤 생성
- 맵 에디터 구현
- 최종 밸런싱
- 동적 가중치/희귀도 메타 완성

## 핵심 결정

### 1. “맵 어디든 랜덤”은 1차에서 하지 않는다

- 벽 안 / 함정 안 / 접근 불가 위치 / 불공정 위치 문제가 크다.
- 따라서 1차는 **맵이 정한 후보 지점 집합** 안에서만 랜덤으로 고른다.

### 2. 고정 스폰과 랜덤 스폰은 공존해야 한다

- 일부 스폰은 전략 포인트로 고정
- 일부 스폰은 후보 중 랜덤
- 둘 다 섞일 수 있게 한다.

### 3. 에디터는 후속 단계다

- 후보 지점 데이터 구조를 먼저 고정하고
- 초기에는 JSON 손편집으로도 충분히 운영 가능하게 한다.

## 완료 조건

- 맵 데이터가 후보 지점 랜덤 스폰을 표현할 수 있다.
- 고정 스폰과 랜덤 후보 스폰을 같은 맵에서 함께 다룰 수 있다.
- 추후 맵 에디터가 붙어도 그대로 사용할 수 있는 구조가 정리된다.

## 검증 방법

- 문서 기준 정합성 검토
- 이후 구현 시 후보 집합에서만 스폰되는지 서버 테스트로 확인
