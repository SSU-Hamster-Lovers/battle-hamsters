# 맵 설계

## 데이터 포맷

**JSON** 사용.

| 장점 | 설명 |
|------|------|
| 범용성 | Phaser (JS) + Rust (serde_json) 모두 호환 |
| 도구 | Tiled 에디터 등이 JSON 지원 |
| 버전관리 | diff로 변경 사항 추적 용이 |

## 기본 구조

```json
{
  "id": "arena_01",
  "name": "기본 아레나",
  "width": 1280,
  "height": 720,
  "spawnPoints": [
    { "x": 100, "y": 360 },
    { "x": 1180, "y": 360 }
  ],
  "terrain": [...],
  "collision": [...],
  "hazards": [...],
  "weaponSpawns": [...],
  "itemSpawns": [...]
}
```

## 구성 요소

| 요소 | 설명 |
|------|------|
| `terrain` | 지형 (바닥, 벽 등) |
| `collision` | 충돌 영역 |
| `spawnPoints` | 스폰 위치 |
| `weaponSpawns` | 무기 스폰 위치 |
| `itemSpawns` | 아이템 스폰 위치 |
| `hazards` | 위험 구역 (낙사 등) |
| `decorations` | 장식 (시각적 요소) |

## 설계 원칙

- 공정성: 스폰 위치 대칭
- 다양성: 다양한 전략 가능
- 스폰킬 방지: 안전한 스폰 위치
- 기동성: 빠른 이동과 점프 루트가 살아 있어야 함
- 순환성: 무기 먹기/버리기/재확보 루프가 자연스럽게 일어나야 함
- 경쟁성: 무기 스폰과 아이템 스폰이 서로 다른 이동 동선을 만들어야 함

## 무기 스폰 원칙

- 맵 스폰 무기는 플레이어 이동 루프를 유도해야 한다.
- 너무 안전한 위치에만 몰아두지 않는다.
- 스폰된 무기와 버려진 무기는 일정 시간이 지나면 사라져야 한다.
- 초기 기준 despawn 시간은 약 **10초**를 목표로 한다.

### 무기 스폰 연출 예시
- 공중 스폰 → 중력 낙하
- 필드 스폰 → 페이드인
- 오브젝트 트리거 스폰 → 코코넛, 상자 등에서 등장

### 무기 디스폰 연출
- 작아지면서 뿅하고 사라지는 연출을 기본 기준으로 둔다.

## 아이템 스폰 원칙

- 아이템 스폰은 무기 스폰과 다른 경쟁 포인트를 만들어야 한다.
- 이동/생존/점프 확장 아이템은 루트 선택을 바꾸는 위치에 배치한다.
- 점프 아이템은 고지대 접근, 우회 루트, 탈출 루트와 연동될 때 가치가 높다.
