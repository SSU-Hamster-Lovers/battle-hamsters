# 무기 설계

## 목표

Battle Hamsters의 전투는 **복잡한 스킬 콤보가 아니라 무기 선택, 위치 선정, 에임, 넉백 관리, 빠른 이동**에서 차이를 만들어야 한다.
무기는 캐릭터 고유 스킬을 대체하는 핵심 전투 수단이며, 플레이어가 맵을 계속 순환하도록 만드는 자원 시스템이어야 한다.

## 전투 감각 원칙

1. **빠른 난전**: 플레이어는 무기를 들고 계속 이동해야 한다.
2. **무기 순환**: 한 무기를 오래 붙잡기보다, 쓰고 버리고 다시 먹는 흐름이 중요하다.
3. **맵 상호작용**: 낙사 구역, 고저차, 좁은 길목, 점프 루트에서 무기 강약이 드러나야 한다.
4. **낮은 입력 복잡도**: 이동 + 조준 + 공격 + 줍기/버리기만으로 충분해야 한다.
5. **데이터 우선**: 수치와 동작 모델을 데이터로 조정 가능해야 한다.

## 전투 슬롯 규칙

- 기본 상태: **맨손**
- 무기 소지 슬롯: **1개**
- 무기 획득: 맵 스폰 또는 바닥 드랍 무기 줍기
- 무기 교체: 현재 무기를 버리고 새 무기 줍기
- 보조 아이템 없음
- 궁극기/액티브 스킬 없음
- 무기 업그레이드 없음

## 무기 시스템 핵심 규칙

### 1. 기본 무기와 보급 무기 구분

- **기본 무기**: 맨손. 무한 사용 가능.
- **보급 무기**: 월드에서 주워서 쓰는 무기. 모두 유한 자원을 가진다.

### 2. 예비 탄약 없음

- 보급 무기는 예비 탄약을 따로 저장하지 않는다.
- 자원이 모두 소진되면 무기는 종료된다.
- 다시 풀 자원 상태로 쓰려면 **새 무기를 주워야 한다.**

### 3. 무기 종료 규칙

- 탄창형 무기는 탄 수가 `0`이 되면 종료
- 빔/용량형 무기는 에너지가 `0`이 되면 종료
- 종료된 무기는 즉시 제거되거나 빈 무기로 버려지는 방식 중 하나를 택할 수 있으나,
  초기 기준은 **전투 흐름을 끊지 않도록 제거 또는 폐기 처리**를 우선한다.

### 4. 월드 무기 라이프사이클

- 맵에서 스폰된 무기와 플레이어가 버린 무기는 모두 월드 오브젝트다.
- 월드 무기는 획득되지 않으면 일정 시간 뒤 사라진다.
- 초기 기준 despawn 시간은 **10초 전후**로 둔다.

## 무기 분류 축

무기는 한 가지 축으로만 분류하지 않고, 아래 조합으로 표현한다.

### 판정 방식

- `melee`
- `hitscan`
- `projectile`
- `beam`

### 발사 방식

- `single`
- `burst`
- `auto`
- `channel`

### 자원 방식

- `infinite` — 기본 무기 전용
- `magazine` — 발당 소모형
- `capacity` — 시간/지속 소모형

이 구조를 쓰면 아래와 같은 조합이 가능하다.

| 무기 예시      | 판정       | 발사        | 자원     |
| -------------- | ---------- | ----------- | -------- |
| Paws           | melee      | single      | infinite |
| Acorn Blaster  | hitscan    | single/auto | magazine |
| Seed Shotgun   | hitscan    | burst       | magazine |
| Berry Launcher | projectile | single      | magazine |
| Laser Cutter   | beam       | channel     | capacity |
| Grab Spear     | melee      | single      | magazine |
| Grenade Pack   | projectile | single      | magazine |

## 이동과 무기 상호작용

이 게임은 이동이 빠르므로, 무기는 정지 사격보다 **이동 중 전투**를 전제로 설계한다.

- 근접 무기: 추격, 낙사 압박, 점프 추격에 유리
- 히트스캔 무기: 중거리 견제에 유리
- 투사체 무기: 예측샷과 지형 장악에 유리
- 빔 무기: 순간 폭딜보다 라인 통제와 지속 압박에 유리

## 조준 각도 / Dead zone 후속

- 현재 MVP는 에임 벡터를 입력으로 보내고, 무기 표현은 주로 좌우 방향 전환 기준으로만 해석한다.
- 후속 단계에서는 무기별로 **허용 발사 각도**와 **시각 오버레이 각도**를 분리해 관리한다.
- 목표는 아래 세 가지다.
  - 이동 방향과 무관한 backpedal 허용
  - 무기별 조준 가능 범위 차별화
  - 상향/하향 조준 시 무기 손 위치 자연화
- 자세한 초안은 `docs/technical/mini-spec-weapon-angle-deadzone-v0.md` 를 기준으로 한다.

## 자기 반동 (`self recoil`) 축

이 게임의 무기는 상대를 미는 넉백만이 아니라,
**발사자 자신이 뒤로 밀리는 자기 반동**도 지원할 수 있어야 한다.

### 기본 원칙

- `knockback`은 **상대에게 주는 힘**
- `selfRecoil`은 **발사자 자신에게 주는 힘**
- 둘은 별개로 조절 가능해야 한다.

### 설계 의도

- 권총류는 자기 반동이 약함
- 핸드캐논/산탄총류는 자기 반동이 커서
  - 공중에서 역추진
  - 낙사 회피/유도
  - 예능성 플레이
    를 만들 수 있음

### 방향 규칙 초안

- 기본은 **현재 에임 방향의 180도 반대**
- 다만 무기별로 아래를 조절 가능해야 한다.
  - 위쪽/아래쪽 각도 보정
  - 소량의 랜덤 각도 오차
  - 지상/공중 배수 차이

## 점프 확장과 무기 메타

- 기본은 1단 점프
- 아이템으로 2단/3단 점프가 열리면, 무기 성능도 달라진다.
- 산탄/근접 무기는 공중 접근과 잘 맞고,
- 빔/히트스캔은 고지대 유지나 추적 압박과 잘 맞는다.

즉, 점프 증가는 단순 이동 강화뿐 아니라 **무기 메타 변화 요소**다.

## Beam 무기 설계 방향

Beam 무기는 단순한 고속 히트스캔 총이 아니라 **지속 조사형 무기**로 본다.

### 기본 원칙

- `hitType = beam`
- `fireMode = channel`
- `resourceModel = capacity`
- 공격 버튼 유지 동안 조사
- 시간당 자원 소모

### 밸런스 축

Beam 무기는 아래 두 방향 모두 가능하다.

1. **사살 유도형**
   - 높은 데미지
   - 낮은 넉백
   - 낙사보다 HP削り에 집중

2. **낙사 유도형**
   - 낮은 데미지
   - 높은 넉백
   - 위치 붕괴와 낙사 압박에 집중

즉, beam이라는 판정 방식은 고정이고, 실제 역할은 데미지/넉백 비율로 갈린다.

## Grab 무기 아이디어

Grab 무기는 상대를 일시적으로 붙잡아 **이동, 낙사 유도, 설치형 무기 연계, 예능성 플레이**를 만드는 특수 무기 계열이다.

### 기본 개념

- 적중 시 일정 시간 동안 상대를 붙잡는다.
- 예시: 꼬챙이 찌르기로 **3초간 들고 다니기**
- 주 용도:
  - 낙사 위치로 운반
  - 설치형/폭발형 무기 연계
  - 혼전에서 강제 위치 이동

### 설계 주의점

- 과도한 군중 제어가 되지 않도록 지속시간은 짧게 유지
- 무적/해제 조건을 명확히 둘 것
- 재미용/하이라이트용 무기 성격이 강함

## 투척 무기 아이디어

수류탄, 지연 신관 폭탄 등 투척형 무기를 고려한다.

### 슬롯 정책

- 투척 무기도 **주무기 슬롯 1칸을 차지하는 무기**로 본다.
- 별도 소모품 인벤토리를 두지 않는다.

이 방식은 Chaos Faction류처럼 로직을 단순하게 유지하는 데 유리하다.

### 장점

- 무기 시스템 통일성 유지
- UI 단순화
- 서버 로직 단순화

즉, 투척 폭탄도 `projectile + magazine` 계열 무기로 취급하는 것이 초기 구조에 적합하다.

## 월드 스폰 연출

무기는 같은 스폰 규칙만 가지는 것이 아니라 **어떻게 나타나는가**도 구분할 수 있다.

### 스폰 연출 예시

- **공중 스폰**: 상공에서 등장 후 중력으로 낙하
- **필드 스폰**: 바닥에 페이드인
- **트리거 스폰**: 코코넛, 상자, 오브젝트를 건드리면 등장

### 디스폰 연출

- 무기가 사라질 때는 **작아지면서 뿅하고 사라지는 연출**을 기본 기준으로 둔다.

## MVP 무기 풀

### 1. Paws (기본 맨손)

- `hitType`: `melee`
- `fireMode`: `single`
- `resourceModel`: `infinite`
- 역할: 무기 없는 상태의 최소 전투 보장
- 의도: 완전히 무력한 상태를 만들지 않되, 월드 무기를 먹을 이유는 남긴다.

### 2. Acorn Blaster

- `hitType`: `hitscan`
- `fireMode`: `single` 또는 저연사 `auto`
- `resourceModel`: `magazine`
- 역할: 가장 표준적인 기본 총기
- 의도: 서버/클라이언트 기본 사격 판정의 기준 무기
- 반동: 낮거나 거의 없음

### 3. Seed Shotgun

- `hitType`: `hitscan`
- `fireMode`: `burst`
- `resourceModel`: `magazine`
- 역할: 근거리 폭발력과 낙사 압박
- 의도: 좁은 지형, 난전, 낙사 맵에서 전략 차별화
- 반동: 강하게 설정 가능

### 4. Hand Cannon (후속 후보)

- `hitType`: `hitscan`
- `fireMode`: `single`
- `resourceModel`: `magazine`
- 역할: 강한 넉백 + 강한 자기 반동으로 이동 자체도 무기화
- 의도: 쏘면 적도 날아가고 나도 날아가는 예능성/숙련도 무기

### 5. Berry Launcher

- `hitType`: `projectile`
- `fireMode`: `single`
- `resourceModel`: `magazine`
- 역할: 투사체 실험용 무기
- 의도: 향후 투사체/폭발 시스템 확장 기준점

## 미래 무기 확장 예시

### Laser Cutter (예시)

- `hitType`: `beam`
- `fireMode`: `channel`
- `resourceModel`: `capacity`
- 특징:
  - 버튼을 누르는 동안 지속 조사
  - 초당 자원 감소
  - 자원 고갈 시 무기 종료
- 의도:
  - 일반 총기와 다른 압박 방식 지원
  - 히트스캔 단발 모델에 억지로 끼워 넣지 않기

### Grab Spear (예시)

- `hitType`: `melee`
- `fireMode`: `single`
- `resourceModel`: `magazine`
- `specialEffect`: `grab`
- 특징:
  - 적중 시 짧은 시간 적을 붙잡아 운반
  - 낙사/설치형 무기 연계 가능

### Grenade Pack (예시)

- `hitType`: `projectile`
- `fireMode`: `single`
- `resourceModel`: `magazine`
- `specialEffect`: `explode`
- 특징:
  - 투척 후 폭발
  - 주무기 슬롯 차지

## 캐릭터 고정 스탯 (MVP)

복잡한 캐릭터 차별화를 피하기 위해 MVP에서는 전 캐릭터 공통 스탯을 사용한다.

- 최대 HP: `100`
- 기본 생명: `3`
- 기본 이동 속도: 빠른 템포를 목표로 추후 조정
- 기본 점프 수: `1`
- 점프 확장: 아이템에 따라 `2` 또는 `3`
- 점프 수 범위: `1 ~ 3`
- 히트박스 크기: 공통
- 무게/넉백 저항: 공통

즉, 초기 메타 차이는 **캐릭터 선택이 아니라 무기, 위치, 점프 상태**에서 발생한다.

## 버프/디버프 규칙 (MVP)

MVP에서는 복잡한 상태이상을 도입하지 않는다.
대신 아래 정도만 유지한다.

- `moveSpeedRank`
- `maxJumpCount` 증가 아이템

미지원 상태 효과:

- 화상
- 중독
- 빙결
- 기절
- 은신
- 지속 피해 디버프

## 무기별 실제 수치표 운영

무기별 실제 수치는 별도 표 형식으로 관리하는 것을 권장한다.

### 추천 표 항목

- 무기명
- `hitType`
- `fireMode`
- `resourceModel`
- 데미지
- 넉백
- 사거리
- 공격 간격
- 최대 자원
- 자원 소모량
- 투사체 속도
- 탄 퍼짐
- 월드 despawn 시간
- 희귀도
- 특수 효과
- 의도된 역할
- 카운터 관계

### 추천 워크플로

1. **AI 1차 검수**
   - 수치표 누락 확인
   - 구조적 밸런스 이상치 탐지
   - 역할 중복/공백 확인
2. **인간 2차 플레이 테스트**
   - 실제 손맛 확인
   - 지나친 사기/무력감 체크
   - 이동 속도와 점프 메타와의 상호작용 검증
3. **수치 재조정**
   - 문서 갱신
   - 구현 반영
