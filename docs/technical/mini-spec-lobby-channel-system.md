# 미니 스펙 — 로비 채널 시스템 (백로그)

## 작업명

로비 채널 시스템

## 목적

로비 허브에 유저가 너무 많이 몰릴 때를 대비해,
같은 유형의 로비 월드맵을 여러 개 운용하고 사용자가 **채널 변경**할 수 있는 구조를 미리 정한다.

## 왜 필요한가

로비 월드맵을 허브형 공간으로 운영하면,
한 맵에 유저가 과도하게 몰릴 수 있다.

그래서 장기적으로는:

- 로비 월드맵 여러 개 운영
- 수용량 초과 시 분산
- 사용자가 채널을 바꾸는 기능
  이 필요할 수 있다.

## 이번 문서 범위

- 채널 개념 정의
- 채널 생성/분산의 기본 아이디어 정리
- 사용자가 채널 변경할 수 있는 UX 후보 정리

## 비목표

- 실제 채널 매니저 구현
- 서버 샤딩 구현
- 친구 따라가기 / 파티 동기화 구현

## 핵심 결정

### 1. 로비 월드는 복수 인스턴스를 가질 수 있다

- 하나의 “로비 종류”에 대해 여러 채널 존재 가능
- 예:
  - Lobby-1
  - Lobby-2
  - Lobby-3

### 2. 수용량 정책은 전투와 다르게 잡을 수 있다

- 로비는 저 TPS 중심
- 플레이 감각보다 수용량/가벼움 우선
- 전투용 룸과 같은 수용량 정책을 강제하지 않는다

### 3. 채널 변경은 사용자가 직접 요청할 수 있어야 한다

후보:

- UI 메뉴에서 선택
- 로비 내 포털/오브젝트 상호작용
- 혼잡 채널 회피용 추천 이동

## 완료 조건

- 로비 채널 구조가 backlog 문서로 정리된다.
- 로비 허브 스펙과 역할이 분리되어 관리된다.
