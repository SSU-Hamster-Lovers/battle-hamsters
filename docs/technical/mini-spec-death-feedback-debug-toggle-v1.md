# 미니 스펙 — 사망 연출 정리 + 라이브 디버그 토글 1차

작업명:
사망 연출 정리 + 라이브 디버그 토글 1차

목표:
리스폰 대기 중 플레이어가 맵 하단으로 순간이동해 깜빡이는 버그를 없애고, 라이브 환경에서 디버그 오버레이를 기본 OFF로 두되 운영자만 숨겨진 경로로 켤 수 있게 한다.

이번 범위:
- `PlayerSnapshot`에 최근 사망 원인 필드 추가
- `fall/hazard` 사망 시 본체 즉시 숨김
- `weapon/self` 사망 시 임시 중력 더미 연출 추가
- 디버그 지형선 / spawn 표식 / collider를 기본 OFF로 전환
- `?ops=1` + 숨겨진 단축키로 디버그 토글 가능하게 구현

건드리는 스펙/문서:
- `docs/technical/data-formats.md`
- `docs/technical/sync-protocol.md`
- `docs/technical/current-implementation.md`
- `packages/shared/world.ts`
- `server/src/*`
- `apps/game/src/main.ts`

비목표:
- 진짜 인증 기반 운영자 권한 시스템
- 완전한 래그돌 물리
- death replay / 관전자 카메라 / 킬캠

검증 방법:
- 무기 처치 시: 본체가 바닥 아래로 순간이동하지 않고, 임시 더미가 짧게 중력 영향을 받는지 확인
- 낙사 / 즉사 함정 시: 본체가 즉시 사라지고 하단 깜빡임이 없는지 확인
- 일반 사용자: 디버그가 기본 OFF인지 확인
- 운영자: `?ops=1` 후 숨겨진 단축키로 디버그 ON/OFF가 가능한지 확인

## 핵심 결정

### 1. 사망 표시
- 서버는 리스폰 대기 플레이어를 임의의 바닥 아래 좌표로 옮기지 않는다.
- 클라이언트는 `lastDeathCause`로 사망 원인을 구분한다.
- `fall_zone`, `instant_kill_hazard`: 플레이어 본체를 즉시 숨긴다.
- `weapon`, `self`: 후속 래그돌 확장을 염두에 두고 1차 임시 중력 더미를 보여준다.

### 2. 디버그 접근 방식
- 현재 프로젝트에는 인증/권한 시스템이 없으므로 “진짜 운영자 전용” 보안은 제공하지 않는다.
- 대신 기본 UI에는 드러내지 않고:
  - `?ops=1` 로 ops 접근을 로컬에 저장
  - 그 상태에서만 숨겨진 단축키로 디버그 토글 허용
- 기본값은 항상 OFF다.

### 3. 추천 단축키
- `Alt + Shift + D`
- 평소 이동 입력과 충돌이 적고, 우연히 눌릴 가능성이 낮다.
