## 1. 플레이어 스폰 모드 초안

### A. 고정 스폰 지점
- 플레이어 인원 제한과 함께 동작한다.
- 각 슬롯의 위치가 고정된다.
- 예:
  - 2인 매치면 고정 스폰 2개만 사용
  - 4인 매치면 고정 스폰 4개 사용

### B. 동적 스폰 지점
- 랜덤 스폰 + 스폰 후보 지점 방식이다.
- 플레이어 인원 제한에 직접 묶지 않는다.
- 프리월드 / 자유맵 / 대규모 맵에서 자연스러운 재진입용으로 적합하다.

## 2. 맵 후보 지점 설계 방향

- 맵 제작 편의를 위해 스폰 후보 지점 하나에 아래 상태를 함께 둘 수 있게 하는 방향을 선호한다.
  - `fixed_spawn` 용 활성 여부
  - `dynamic_spawn` 용 활성 여부
- 즉, 동일한 후보 지점을:
  - 고정 스폰으로만 사용
  - 동적 스폰으로만 사용
  - 둘 다 사용
  - 둘 다 사용 안 함
  으로 설정할 수 있게 하는 방향이다.

### 기대 효과
- 맵 제작자가 지점을 이중으로 만들 필요가 줄어든다.
- 같은 맵이라도 프리월드 / 매치 / 특수 룰에 따라 재사용이 쉬워진다.

## 3. 무기 pickup 표현 방향

- 현재 임시 표현은 단색 타원 + 라벨이다.
- 후속 목표는 메탈 슬러그식 “무기 꾸러미 + 식별 코드/상징” 방향이다.
- 후보:
  - 무기 꾸러미 sprite
  - 전용 픽토그램
  - 짧은 철자 코드
  - 전용 폰트 또는 sprite glyph

### 표현 규칙 메모
- pickup 은 “맵 위에 놓였을 때 무엇인지 한눈에 알아볼 수 있어야 한다”.
- sprite 가 아직 없을 때도 코드/기호만으로 식별 가능해야 한다.
- 무기 종류가 늘어도 같은 UI 문법 안에서 확장 가능해야 한다.

## 4. 장착 표현 방향

- 햄스터가 무기를 주우면, 손에 해당 무기를 들고 있는 모습이 보여야 한다.
- 이는 후속 구현에서 반드시 고려해야 하는 항목으로 잡는다.
- 최소 1차 목표:
  - 기본 본체 sprite
  - 장착 무기 sprite/overlay
  - 방향(`left/right`)에 따른 반전

## 5. 발사 표현 방향

- 현재는 발사 방향/각도를 선형 임시 효과로 보여준다.
- 후속에는 무기 종류별로 발사 연출을 분리한다.

### 후보 규칙
- `Paws`
  - 원이 점점 커지는 근접 충격파형 이펙트
- 총알 계열
  - 총구 화염
  - 탄 궤적 또는 짧은 발사 flash
- 레이저 계열
  - 선형 beam 연출 유지 가능
  - 대신 색/두께/끝단 효과는 무기별 차별화

## 6. 후속 구현 시 분해 포인트

1. 맵 JSON / shared 타입에 스폰 모드 필드 추가
2. 룸 타입/모드에 따라 `fixed` / `dynamic` 스폰 해석 분기
3. pickup sprite + 코드 glyph 시스템 정의
4. 햄스터 본체와 무기 overlay 렌더 구조 분리
5. 무기별 발사 VFX registry 도입
