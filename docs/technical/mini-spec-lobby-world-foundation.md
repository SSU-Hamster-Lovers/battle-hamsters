# 미니 스펙 — 로비 월드맵 기반 허브 (백로그)

## 작업명

로비 월드맵 기반 허브

## 목적

딱딱한 텍스트 로비 대신, 사용자가 로그인하면 즉시 들어가서 뛰어놀 수 있는
**저 TPS / 고수용 로비 월드맵 허브**의 기준을 미리 정한다.

## 왜 필요한가

장기적으로는 Portal을 단순 메뉴가 아니라,
플레이어가 실제로 존재하는 공간형 허브로 확장할 수 있다.

예시:

- 로그인 즉시 로비 월드맵에 스폰
- 다른 유저와 가볍게 뛰어놀기
- 매칭 진입 / 채널 이동 / 상점 / 상호작용 확장

이 구조는 전투 서버와 별개로
**허브형 경험 + 사회적 공간 + 시스템 진입점**을 한 번에 묶을 수 있다.

## 이번 문서 범위

- 로비 월드맵 허브의 장기 개념 정의
- 전투 서버와의 역할 분리
- 허브에서 가능한 행동 후보 정리
- 나중에 채널 시스템과 연결될 수 있는 구조 초안

## 비목표

- 즉시 구현
- 실제 상점 시스템
- 실제 채널 전환 구현
- 실제 대규모 동접 최적화 구현

## 핵심 결정

### 1. 로비는 “텍스트 메뉴”가 아니라 “공간형 허브”가 될 수 있다

- 로그인하면 실제 월드맵에 스폰
- 이동 / 점프 / 간단한 상호작용 가능
- 전투와 별개의 저부하 공간

### 2. 로비 월드맵은 전투 서버와 다르게 운영할 수 있다

- 낮은 TPS
- 높은 동시 수용
- 전투 판정보다 이동/표시/허브 기능 우선

즉, 로비는 전투 아레나와 다른 성격의 서버/룸 정책을 가질 수 있다.

### 3. 허브 기능은 단계적으로 열린다

후보:

- 자유 이동
- 매칭 진입
- 채널 이동
- 상점/NPC
- 꾸미기/프로필
- 튜토리얼 입구

## 후속 연결 포인트

### 매치 종료 후 프리월드 이동

매치룸에서 `Finished` 상태가 되면 현재는 결과 화면만 보여주고 방이 유지된다.
향후 로비 월드맵이 구현되면:
- 매치 종료 → 결과 표시 → 자동으로 프리월드(자유맵 허브)로 이동
- 클라이언트가 매치룸 WS 연결을 끊고 프리월드 WS 에 재접속
- 또는 서버가 프리월드 roomId 를 결과 메시지에 포함해 클라가 자동 전환

이 기능은 프리월드 허브가 실제로 구현된 이후에 진행한다.

## 완료 조건

- 로비 월드맵 허브 개념이 문서로 고정된다.
- 전투 로드맵과 섞이지 않고 backlog로 분리된다.
- 나중에 채널/상점/매칭 스펙과 연결 가능한 기준선이 생긴다.
